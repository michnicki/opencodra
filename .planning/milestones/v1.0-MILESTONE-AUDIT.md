- **Flow 1 — New Bitbucket repo onboarding → first review**: traced end-to-end through live source. `POST /api/repos/bitbucket` (Phase 6) → transactional repo+credential insert → `POST /webhook/bitbucket` (Phase 5) → HMAC verify via Phase 4's stored credential → `findRepositoryByBitbucketIdentity` resolves the just-created repo → `core/webhook-ingest.ts` (Phase 3) enqueues the job → `core/review.ts` resolves `VcsService.forRepo` → `BitbucketAdapter` (Phase 2 interface / Phase 5 implementation) → diff fetch, inline findings, Code Insights report, build status. GitHub and Bitbucket OAuth login are fully separate route handlers/allow-lists with no shared mutable state. **WIRED, no regression.**
- **Full regression suite re-run live during this audit**: `npm test` → 44 files / 387 tests passed; `npm run typecheck` → clean (0 errors).
