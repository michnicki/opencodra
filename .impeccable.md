## Design Context

### Users
Developers and DevOps leads who manage automated GitHub PR review pipelines. They're power users who visit the dashboard to monitor job status, debug failures, inspect AI-generated code review findings, and understand system health. Context of use: professional, focused, often debugging something. They value clarity and information density over decoration.

### Brand Personality
**Precise. Understated. Dependable.**

Codra is a tool that works quietly in the background, earning trust through reliability. The interface should feel like a well-calibrated instrument — unobtrusive but always accurate. Not flashy, not corporate, not "startuppy." More like a Leica camera than an iPhone. More like a Bloomberg terminal with good taste than a typical SaaS dashboard.

### Aesthetic Direction
- **Theme**: Both light and dark modes, with thoughtful implementation of each
- **Visual tone**: Refined minimalism — generous whitespace, strong typographic hierarchy, deliberate restraint. Every element earns its place.
- **Color**: Cool-leaning slate neutrals tinted toward the brand hue. A single trustworthy indigo-slate accent (`oklch(42% 0.14 258)` in light, `oklch(70% 0.12 258)` in dark). Semantic colors for status (success/warning/danger) using OKLCH for perceptual uniformity.
- **Typography**: Figtree (Google Fonts) — humanist sans-serif that reads as warm but precise. JetBrains Mono for code blocks.
- **Anti-references**: Not the "dark mode + neon accents" AI tool look. Not glassmorphism everywhere. Not the hero metric template (big number + color blob + icon). Not Inter + purple gradients.

### Design Principles
1. **Clarity over cleverness** — Information should be immediately legible. Hierarchy does the work, not decoration.
2. **Restraint is a feature** — Use color sparingly so it carries meaning. A red badge matters because everything else is neutral.
3. **System over novelty** — Consistent spacing rhythm, consistent token use, consistent interaction patterns. Users should never have to learn the same pattern twice.
4. **Motion with purpose** — Smooth state transitions, staggered page entrance. Never animate just to animate.
5. **Trust through density** — Pack information confidently. Don't hide things behind clicks. Developer tools users want to see the data.
